## 1. 软件概述

### 1.1 支持的协议

| 协议 | 模式 | 说明 |
|------|------|------|
| **TCP 客户端** | 主动连接 | 连接到远程 TCP 服务器，支持多实例同时连接 |
| **TCP 服务端** | 监听模式 | 监听本地端口，接收多个客户端连接 |
| **UDP** | 无连接通信 | 发送/接收 UDP 数据报 |
| **串口** | 物理串口通信 | 支持多串口同时连接，波特率 1200~921600 |
| **WebSocket 客户端** | 主动连接 | 连接到 WebSocket 服务器 |
| **WebSocket 服务端** | 监听模式 | 监听本地端口，接收 WebSocket 连接 |
| **MQTT 客户端** | 发布/订阅 | 独立的 MQTT 窗口，支持 TLS、遗嘱消息 |

### 1.2 多协议并发

所有协议可以**同时连接**，互不干扰。例如可以同时连接：
- TCP 客户端 连接 3 个不同服务器
- TCP 服务端 监听 2 个不同端口
- 2 个串口
- UDP、WebSocket

## 3. 通信连接

### 3.1 TCP 客户端

**操作步骤：**

1. 协议选择框选择 `TCP 客户端`
2. 填写目标 **IP** 和 **端口**
3. （可选）勾选 **Keepalive** 并设置参数：
   - **空闲(秒)**：无数据后多久开始发送探测包（默认 60）
   - **间隔(秒)**：探测包发送间隔（默认 10）
   - **次数**：连续未回复次数后断开（默认 5）
4. 点击 **▶ 新增连接** 添加到连接详情表格
5. 点击表格中该行，或点击 **▶▶ 全部连接** 发起连接

> **多实例支持**：可以同时连接多个不同的 TCP 服务器，
> 每次输入不同的 IP:端口后点击"新增连接"即可。

### 3.2 TCP 服务端

**操作步骤：**

1. 选择 `TCP 服务端`
2. 填写 **IP**（本机地址，如 0.0.0.0）和 **端口**
3. 点击 **▶ 新增连接** 添加到表格
4. 在表格中点击该行，点击 **↻ 连接选中** 开始监听

> 每个端口可以单独启动监听，多个端口同时监听。
> 客户端连接时会自动出现在表格的子节点中。

### 3.3 UDP

**操作步骤：**

1. 选择 `UDP`
2. 填写目标 **IP** 和 **端口**
3. **本地端口**：填 `0` 表示系统自动分配，固定值则接收回复到该端口
4. 点击 **▶ 新增连接**

### 3.4 串口

**操作步骤：**

1. 选择 `串口`
2. 选择串口设备，设置**波特率**、**数据位**、**校验**、**停止位**
3. 点击 **▶ 新增连接**
4. 点击表格中该行连接

> **多串口支持**：可以同时连接多个不同的串口设备。

### 3.5 WebSocket 客户端

**操作步骤：**

1. 选择 `WebSocket 客户端`
2. 填写服务器 **IP** 和 **端口**
3. 点击 **▶ 新增连接**

### 3.6 WebSocket 服务端

**操作步骤：**

1. 选择 `WebSocket 服务端`
2. 填写监听 **IP** 和 **端口**
3. 点击 **▶ 新增连接**

### 3.7 MQTT 客户端

MQTT 是一个独立窗口，通过菜单 `工具 → MQTT 客户端` 打开。

包含 6 个标签页：

| 标签页 | 功能 |
|--------|------|
| **连接配置** | Broker 地址、端口、用户名密码、TLS、遗嘱消息 |
| **发布消息** | 发布到指定主题，支持定时发送 |
| **订阅管理** | 订阅/取消订阅主题 |
| **消息转发** | MQTT 消息转发到串口/TCP/UDP |
| **脚本触发** | 收到匹配主题时执行命令、写文件、发 HTTP 请求 |
| **诊断工具** | Broker 信息、Ping 延迟、协议抓包 |

## 4. 连接管理

### 4.1 连接详情表格

通信设置面板下方显示了所有已添加的连接记录：

| 列 | 说明 | 操作 |
|----|------|------|
| **发送** ☑/☐ | 是否从此连接发送数据 | 点击切换 |
| **接收** ☑/☐ | 是否接收此连接的数据 | 点击切换 |
| **协议** | 协议类型 | - |
| **地址** | 目标地址 | - |
| **状态** | 🟢 已连接 / 🔴 已断开 | - |
| **TX** | 发送字节数 | - |
| **RX** | 接收字节数 | - |

> TCP 服务端有客户端连接时，会在父节点下显示子节点（客户端列表）。

### 4.2 批量操作按钮

| 按钮 | 功能 |
|------|------|
| **▶ 新增连接** | 将当前参数添加到表格 |
| **▶▶ 全部连接** | 连接表格中所有已断开的记录 |
| **■ 全部断开** | 断开所有已连接的记录 |
| **✕ 断开选中** | 断开当前选中的行 |
| **↻ 连接选中** | 连接当前选中的行 |
| **✂ 清空选中** | 从表格中删除选中的行 |
| **🗑 清空全部** | 清空表格中所有记录 |
| **☑/☐ 全部发送/取消** | 批量开关所有连接的发送开关 |
| **☑/☐ 全部接收/取消** | 批量开关所有连接的接收开关 |

### 4.3 右键菜单

在连接详情表格中右键，可以断开选中的连接。

## 5. 收发日志

### 5.1 日志显示

日志面板显示所有收发的数据和系统信息，每行格式：
```
[时间戳] 方向 >> 数据内容
```

- **TX**（蓝色）：发送的数据
- **RX**（绿色）：接收的数据
- **INFO**（橙色）：系统信息、告警等

### 5.2 显示模式

| 模式 | 说明 |
|------|------|
| **Hex** | 十六进制显示 |
| **ASCII** | 文本显示 |
| **Hex+ASCII** | 混合显示（默认） |

### 5.3 日志过滤

| 功能 | 位置 | 说明 |
|------|------|------|
| **级别过滤** | 左侧下拉框 | 所有级别 / 信息 / 警告 / 错误 |
| **方向过滤** | 左侧下拉框 | 全部 / 仅 TX / 仅 RX |
| **关键词搜索** | 搜索框 | 输入关键词，▲ ▼ 跳转，✕ 清除 |
| **自动滚动** | 复选框 | 自动滚动到最新日志 |

### 5.4 浮动窗口

点击 **➕新窗口** 可以创建独立的浮动日志窗口，每个浮动窗口有独立的过滤和显示设置，数据与主窗口同步。

### 5.5 自动保存

1. 点击工具栏 **💾 自动保存:关** 按钮
2. 选择保存目录
3. 之后所有收发数据自动追加到日期文件中

### 5.6 日志导出

**位置：** 左侧导航 → 🛠 辅助工具 → 导出

| 格式 | 说明 |
|------|------|
| **TXT** | 纯文本格式 |
| **CSV** | 表格格式，可用 Excel 打开 |
| **HTML** | 带颜色高亮的网页格式 |
| **Excel** (.xlsx) | 带格式的 Excel 表格 |
| **PDF** | 便携文档格式 |

支持范围选择：全部日志、仅 TX、仅 RX。

## 6. 工具栏（全部工具）

### 6.1 发送

#### 快捷发送
- Hex/ASCII/混合模式选择
- 编码选择（UTF-8、GBK 等）
- 前缀/后缀/行延迟
- 发送历史记录
- 定时自动发送
- Ctrl+Enter 快速发送

#### 文件发送
- 选择文件按二进制发送
- 可配置块大小、块间延迟
- 可选 4 字节长度头 + CRC16 尾部

#### 协议解析
- 加载协议模板字段定义
- 支持**多协议同时加载**（HEX + JSON 混合）
- 勾选 **⚡自动解析** 后，收到数据自动检测并解析
- 自动检测数据格式：JSON 文本 → JSON 协议解析，HEX 数据 → HEX 协议解析
- 支持 HEX 编码的 JSON 数据自动解码识别
- HEX 解析结果表格显示每个字段的期望值、实际值、比对结果
- JSON 解析结果表格显示键路径、期望类型、约束、实际值、比对状态
- 支持固定值、长度、校验、枚举映射、位解析等 HEX 字段类型
- 支持 JSON Schema 约束比对：类型、枚举、范围（最小值/最大值）、正则表达式

#### 协议编辑器
- 树形浏览预设/自定义协议模板
- 支持 HEX 和 JSON 两种协议类型，`[H]`/`[J]` 前缀区分
- 可视化编辑协议字段
  - HEX 字段：字段名、Hex 值、字节数、类型、枚举映射、位标志
  - JSON 字段：键路径、类型、必填、默认值、枚举、最小值、最大值、正则
- 支持自动计算长度和校验字段
- JSON 数据生成、格式化、压缩、验证
- 切换命令时自动保存当前编辑的字段
- 自定义协议和命令掉电保存到配置文件
- 导入/导出模板

### 6.2 数据处理

#### 转换器
Hex / 十进制 / 二进制 / ASCII 互转，支持大小端切换。

#### 校验和
支持 CRC8、CRC16-Modbus、CRC16-CCITT、CRC32、MD5、SHA1、SHA256、XOR 校验、和校验等。

#### 格式化
JSON/XML 自动检测与格式化，支持从 HEX 解码。

#### 编码
UTF-8、GBK、Big5、Shift-JIS 等多编码转换。

#### Base64
Base64 编码/解码，支持 URL 安全模式。

### 6.3 测试

#### 压力测试
- 多种数据模式：固定/递增/递减/随机/循环
- 发送次数、间隔、超时设置
- 实时统计（发送/成功/失败/速率）
- 进度条

#### 脚本自动化
- 内置 Python 脚本引擎
- 支持的 API：

| API | 说明 |
|-----|------|
| `send(data)` | 发送数据（bytes 或 hex 字符串） |
| `wait(ms)` | 等待指定毫秒 |
| `log(msg)` | 输出日志 |
| `assert_eq(a, b)` | 断言相等 |
| `assert_contains(s, sub)` | 断言包含 |
| `hex_to_bytes(s)` | Hex 字符串转 bytes |
| `bytes_to_hex(b)` | bytes 转 Hex 字符串 |

- 支持脚本文件的新建、打开、保存
- 运行时可停止

#### 定时任务
在指定时间（HH:MM:SS）定时发送数据，支持每日重复。

#### 录制
录制操作步骤（发送/接收），可按时间回放，导出为 Python 脚本。

#### 随机数据
生成随机 HEX/ASCII/数字/混合数据，支持自定义模板格式。

### 6.4 监控

#### 告警
设置条件规则，匹配时触发弹窗或日志告警。

| 匹配模式 | 说明 |
|----------|------|
| 精确匹配 | 数据完全等于指定内容 |
| 包含 | 数据包含指定内容 |
| 开头匹配 | 数据以指定内容开头 |
| 结尾匹配 | 数据以指定内容结尾 |

#### 波形
实时显示接收数据字节的波形图，支持自动/手动 Y 轴范围。

#### 统计
TX/RX 包数、字节数、错误数、速率统计（1 分钟滚动窗口）。

#### 嗅探
自动识别 Modbus RTU/TCP、EtherNet/IP、MQTT、HTTP 等协议特征。

#### 回放
记录收发数据，支持单条/选中/全部/循环回放，导入/导出 JSON。

#### 对比
逐字节或逐行比对两段 HEX 数据，差异高亮显示。

### 6.5 辅助工具

#### 正则
正则表达式测试，支持常用模式快捷输入和替换功能。

#### 时间戳
Unix 时间戳与日期时间互转，支持多种格式和时区。

#### ASCII 表
0-255 ASCII 码表查询，支持过滤控制字符和扩展字符。

#### 位操作
位与、位或、位异或、位取反、左移、右移，支持 8/16/32/64 位。

#### 备注
添加时间戳备注到日志，支持颜色标记和快速备注按钮。

#### 导出（日志）
见 [5.6 日志导出](#56-日志导出)

### 6.6 网络

#### 隧道
TCP→TCP、TCP→UDP、UDP→TCP、UDP→UDP 双向数据转发。

#### 心跳
收到特定数据时自动回复，或定时发送心跳包。

#### 自动回复
多规则自动回复，支持精确/包含/开头/结尾匹配模式。

## 9. 工程文件

### 9.1 保存工程

`文件 → 📁 保存工程`（Ctrl+S）

保存为 `.cdt` 文件，包含：

- 通信参数设置（协议、IP、端口、串口等）
- 连接详情表格中的所有记录
- 所有工具面板的设置（发送历史、压力测试参数等）
- 快捷键数据
- 全局设置（主题、字体、日志限制等）
- 窗口大小和分割比例
- 当前选中的工具面板

### 9.2 加载工程

`文件 → 📂 加载工程`（Ctrl+O）

选择之前保存的 `.cdt` 文件，恢复所有状态。

> 加载时会先清空当前的所有连接记录和工具面板缓存，所以加载后恢复到保存时的完整状态。

## 10. 软件更新

### 10.1 检查更新

- **自动检查**：启动后 3 秒自动检测
- **手动检查**：`帮助 → 检查更新`

### 10.2 更新流程

1. 检测到新版本后弹出更新对话框
2. 显示版本号、系统版本、更新内容、即将下载的包名
3. 点击 **⬇ 下载** 开始下载
4. 下载完成后询问是否安装
5. 确认后自动安装并重启

### 10.3 忽略版本

点击 **忽略此版本** 后，该版本不会再提示更新。后续发布更高版本时仍会提示。

### 10.4 更新源

支持多源检测：优先通过 Gitee API 检测，API 受限时自动降级到网页抓取。

- 协议解析面板新增 JSON 协议解析
  - JSON 解析结果表格展示：键路径/期望类型/约束/实际值/比对状态
  - 支持 JSON Schema 约束比对（类型/枚举/范围/正则）
  - 自动检测 JSON/HEX 数据格式
  - 支持 HEX 编码的 JSON 数据自动解码
  - 选择协议对话框：全选/全不选、右键折叠/展开、总复选框联动子项
- 自定义协议和命令掉电保存到 config.json

**bug 修复：**
- 新建 JSON 协议后树形显示 `[H]` 而非 `[J]` 的问题

## 12. 常见问题

### Q: 无法连接串口，提示 Resource busy

A: 串口被其他程序占用了。关闭其他串口工具后重试，或在终端运行以下命令查看占用：
```bash
lsof /dev/cu.xxx
```

### Q: 发送数据没有反应

A: 检查连接详情表格中该连接的"发送"开关是否已勾选（☑）。

### Q: 程序打开报 zsh: killed

A: macOS 安全限制。右键 → 打开，或在终端运行：
```bash
xattr -d com.apple.quarantine /路径/到/CommDebugTool
```

### Q: 导出的 PDF 中文显示为方块

A: 确保系统已安装中文字体。macOS 使用 `Arial Unicode.ttf`，Windows 使用 `SimSun`，Linux 需要安装 `fonts-wqy-microhei`。

### Q: 如何恢复出厂设置？

A: 关闭软件后，删除程序目录下的 `config.json` 文件，重新启动即可恢复到默认配置。

### Q: 串口下拉框中出现重复的端口？

A: macOS 上同一 USB 串口会同时注册 `cu.usbserial-*` 和 `cu.wchusbserial*` 两个名称。选择其中一个即可，功能一样。
